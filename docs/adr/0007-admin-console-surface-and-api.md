# Admin Console Is a Same-Origin Surface With Its Own Adapter API

The **Admin Console** is served by the **Aggregated Runtime** as a same-origin bundled SPA (Vue 3 + Vite, in a `web/` workspace package) mounted at `/admin`, backed by a dedicated `/admin/api` surface authorized only by an **Admin Session** plus CSRF. We keep this separate from the existing `/api/v1` rather than extending it: `/api/v1` is a versioned, bearer-authenticated, tag-scoped contract for CLI and MCP clients, while the Admin Console needs full-admin visibility, cookie/CSRF semantics, and write actions. Folding both trust models into one surface would entangle a public contract with admin-only mutation. Because `/admin/api` is a fresh surface with cookie + same-origin + CSRF protection from birth, the existing `rejectBrowserOriginRequests` guard on `/api/v1` (and the CLI token route) is left untouched — browsers are still distrusted there.

`/admin/api` is strictly an HTTP adapter over the domain services that already own each operation — **Config Change** (server enable/disable, reload, with backup and reload observation), **Server Installation Workflow**, the preset domain, and the backend OAuth flow. Route handlers carry no business logic and never write configuration directly, so backups, validation, and hot-reload are never bypassed.

The legacy `/oauth` dashboard is merged into the console as a section rendered from `/admin/api`, but the OAuth flow endpoints that carry registered redirect URIs (`/oauth/authorize/:serverName`, `/oauth/callback/:serverName`) remain at their current paths to avoid forcing every backend server to re-authorize; only their human-facing success/error redirects are repointed into `/admin`, and the old dashboard HTML is retired.

The SPA is embedded into the SEA binary as brotli-precompressed SEA assets served with content-encoding negotiation, falling back to `express.static('web/dist')` when running from source (detected via `node:sea` `isSea()`). This keeps the binary lean and avoids the ~33% penalty of base64-inlining assets into the JS bundle.
